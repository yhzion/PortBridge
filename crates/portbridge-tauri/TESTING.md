# PortBridge (Tauri) — 수동 E2E 검증 & macOS 패리티 체크리스트

Epic #104 · Phase5(S5, #110). 자동화 E2E는 별도 트랙(out-of-scope). 이 문서는 **사람이 실제
데스크톱 앱을 띄워** 핵심 플로우를 검증하고 macOS 원본(`apps/macos`)과 시각·동작 패리티를
확인하기 위한 회귀 방지 체크리스트다.

> ⚠️ 헤드리스/CI로는 트레이·팝오버·autostart·실제 ssh 포워딩의 런타임 동작을 검증할 수 없다
> (CI `tauri-build`는 ubuntu/windows `cargo build`만). 아래 항목은 **macOS GUI 세션에서 수동 수행**한다.

## 실행

```bash
cd crates/portbridge-tauri
npm install
npm run tauri dev      # 개발(핫리로드) — 트레이/팝오버 포함 실제 창
# 또는
npm run tauri build    # 릴리즈 번들
```

전제: 검증용 원격 서버 1개 이상(ssh 키 인증 가능), `~/.ssh/config` alias(선택), 원격에 `ss` 또는 `lsof`.

## A. 핵심 플로우 E2E 시나리오

각 항목: 단계 → 기대결과. 실패 시 갭으로 기록(§D).

### A1. 앱 시작 · 트레이 · 팝오버 (S4)

- [ ] 앱 실행 → 메뉴바에 트레이 아이콘 표시, 창은 **숨김** 상태로 시작.
- [ ] 트레이 아이콘 **좌클릭** → 팝오버 창 표시(+포커스). 다시 좌클릭 → 숨김(토글).
- [ ] 팝오버 표시 중 **바깥 클릭(blur)** → 자동 숨김.
- [ ] 트레이 **우클릭** → 메뉴(열기/종료). "열기"→창 표시, "종료"→앱 종료.
- [ ] 앱 **재실행**(이미 실행 중) → 새 인스턴스가 뜨지 않고 기존 창 포커스(single-instance).

### A2. 서버 추가 (AddServerModal)

- [ ] 빈 상태에서 "＋ 서버 추가" → 모달(이름/사용자/호스트/포트, 포트 기본 22).
- [ ] 사용자/호스트 비우면 "추가" 비활성. 포트 1–65535 밖이면 검증 에러.
- [ ] 동일 (user, host, port) 중복 입력 → 중복 에러로 추가 차단.
- [ ] 유효 입력 + 추가 → 목록에 서버 섹션 생성 + **자동 스캔** 시작.

### A3. 포트 스캔 (scanState)

- [ ] 추가/시작 시 섹션이 "스캔 중…" → 완료 시 포트 목록(loaded).
- [ ] 헤더 ↻(전체)·섹션별 ↻ → 재스캔 동작.
- [ ] 포트 없음 → "포워딩되지 않은 포트 없음".
- [ ] (현재 한계) 오프라인/도구없음/인증실패는 평탄 에러로 표시 — 구조화 분류는 #125.

### A4. 포워딩 토글

- [ ] idle 포트 행의 토글 → "포워딩 연결 중…"(Starting) → 성공 시 "포워딩 중"(Active),
      상단 "포워딩 중" 섹션으로 이동 + 활성 카운트 배지 증가.
- [ ] Active 행 토글 → 중지, 섹션에서 사라지고 다시 idle 포트로 노출.
- [ ] "브라우저에서 열기"(Active) → 로컬 포트가 브라우저로 열림.

### A5. 포트 충돌 해소 (PortConflictModal)

- [ ] 이미 사용 중인 로컬 포트로 포워딩 시도 → "로컬 포트 N이(가) 사용 중입니다" 모달.
- [ ] 입력 기본값 = N+1, N과 같으면 "연결" 비활성. 다른 포트 입력 → 연결 → 포워딩 성공.

### A6. 즐겨찾기

- [ ] 포트 행의 별 토글 → 채워짐/비워짐 즉시 반영(낙관적), 재시작 후에도 유지(영속).

### A7. 서버 편집 / 삭제

- [ ] 섹션 ⋯ 메뉴 → "편집…" → 필드 프리필, 저장 시 갱신 + 재스캔.
- [ ] 접속정보(user/host/port) 변경 시 동작 확인(현재: 단순 재스캔; macOS의 활성 터널 재시작은 후속).
- [ ] ⋯ → "삭제" → 확인 다이얼로그(활성 포워딩 있으면 종료 경고 문구) → 삭제 시 활성 터널 정리 + 섹션 제거.

### A8. 검색 / 펼침 / 일괄

- [ ] 검색창에 포트번호/프로세스명 → 매칭 행만 필터. 무결과 → "'…'에 일치하는 결과가 없습니다".
- [ ] "모든 서버" 헤더의 모두 펼치기/접기 토글.
- [ ] "포워딩 중" 헤더의 "모두 끄기" → 활성 전부 중지.

### A9. 에러 토스트

- [ ] 백엔드 에러 발생 → 하단 빨강 토스트(최대 3개), **약 5초 후 자동 소멸**, ✕로 즉시 닫기.

### A10. 테마

- [ ] 헤더 "테마" 토글로 자동/라이트/다크 순환 → 즉시 반영.
- [ ] "자동"에서 OS 외관 변경 시 실시간 추종.

### A11. autostart (현재 와이어 수준)

- [ ] 플러그인 초기화 확인(앱 정상 기동). 토글 UI는 미구현(#133) — 수동 확인 불가, 후속.

## B. macOS 패리티 (시각/동작 비교)

`apps/macos` 원본을 동일 데이터로 띄우고 Tauri 앱과 **라이트/다크 각각** 비교. 화면별로 캡처해
`docs/` 또는 PR에 첨부하고 차이를 §D에 기록.

| 화면              | 비교 포인트                                                                            |
| ----------------- | -------------------------------------------------------------------------------------- |
| 빈 상태           | 아이콘·문구·"서버 추가" 버튼 위치                                                      |
| 서버 섹션         | 모노그램(색/이니셜)·primary(name??host)/secondary(user@host[:port])·활성 배지·재스캔·⋯ |
| 포워딩 행         | idle(포트+프로세스)/Active("→ :local 포워딩 중")/Starting(스피너)/Error 배지           |
| 활성 섹션         | "포워딩 중 · N" + "모두 끄기"                                                          |
| AddServer 모달    | 필드 그룹·플레이스홀더·버튼                                                            |
| PortConflict 모달 | 제목·안내·입력·취소/연결                                                               |
| 에러 토스트       | 빨강 배경/아이콘/문구                                                                  |
| 트레이/팝오버     | 아이콘·창 위치·blur 숨김                                                               |

알려진 시각 차이(후속): 모노그램 톤 농도, 트레이 아이콘(현재 앱아이콘 template), 팝오버 위치(현재 center) — #133.
